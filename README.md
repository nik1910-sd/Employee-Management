# 🧑‍💼 Employee Management System

A RESTful API built with **Spring Boot** and **Spring Data JPA** for managing employee records. This project was built as a **hands-on practice project** to learn and implement core Spring Boot concepts including JPA, Hibernate, MySQL integration, and REST API development.

---

## 🎯 Purpose

This project was created as part of a Spring Boot learning journey to practice:
- Building REST APIs with Spring Boot
- Using Spring Data JPA with Hibernate for database operations
- Connecting a Spring Boot application to a MySQL database
- Implementing a clean layered architecture (Controller → Service → Repository)
- Understanding JPA entity annotations and how they map to database tables

---

## ✨ Features

- ➕ Add a new employee
- 📋 Fetch all employees
- 🔍 Fetch a single employee by ID
- ✏️ Update an existing employee
- 🗑️ Delete an employee
- ⚡ Auto table creation via Hibernate (`ddl-auto=update`)
- 🛡️ Exception handling for employee not found scenarios

## ⚙️ Setup & Installation

### Prerequisites

Make sure you have the following installed:
- [Java 21](https://www.oracle.com/java/technologies/downloads/#java21)
- [Maven](https://maven.apache.org/download.cgi)
- [MySQL](https://dev.mysql.com/downloads/mysql/) (running on port 3306)
- [Postman](https://www.postman.com/downloads/) (for testing APIs)

### Step 1 — Clone the Repository

```bash
git clone https://github.com/nik1910-sd/Employee-Management.git
cd Employee-Management
```

### Step 2 — Create MySQL Database

Open MySQL and run:

```sql
CREATE DATABASE jpa_basic_emp;
```

> ✅ You don't need to create the `employees` table manually — Hibernate will create it automatically when the app starts (`ddl-auto=update`).

### Step 3 — Configure Database Credentials

Open `src/main/resources/application.properties` and update your MySQL credentials:

```properties
spring.datasource.url=jdbc:mysql://localhost:3306/jpa_basic_emp
spring.datasource.username=root
spring.datasource.password=root        # change this to your MySQL password
```

### Step 4 — Run the Application

```bash
mvn spring-boot:run
```

Or run `EmployeeMngtApplication.java` directly from your IDE (IntelliJ IDEA / Eclipse).

The app will start on: **`http://localhost:8080`**

## 📡 API Endpoints

Base URL: `http://localhost:8080/employees`

| Method | Endpoint | Description | Request Body |
|---|---|---|---|
| `POST` | `/employees` | Add a new employee | JSON (see below) |
| `GET` | `/employees` | Get all employees | None |
| `GET` | `/employees/{id}` | Get employee by ID | None |
| `PUT` | `/employees/{id}` | Update employee by ID | JSON (see below) |
| `DELETE` | `/employees/{id}` | Delete employee by ID | None |

### Request Body (POST / PUT)

```json
{
  "name": "Rahul Sharma",
  "email": "rahul@example.com",
  "department": "Engineering",
  "salary": 75000.0
}
```

### Sample Response

```json
{
  "id": 1,
  "name": "Rahul Sharma",
  "email": "rahul@example.com",
  "department": "Engineering",
  "salary": 75000.0
}
```

### Error Response (Employee Not Found)

```json
{
  "message": "Employee not found by id = 99"
}
```

## 🗄️ Database Schema

The `employees` table is auto-generated by Hibernate:

```sql
CREATE TABLE employees (
  id          INT PRIMARY KEY AUTO_INCREMENT,
  name        VARCHAR(30)  NOT NULL,
  email       VARCHAR(50)  NOT NULL,
  department  VARCHAR(50)  NOT NULL,
  salary      DOUBLE       NOT NULL
);
```

---

## 🔑 Key Concepts Practiced

- **`@Entity` / `@Table`** — Map Java class to MySQL table
- **`@Id` / `@GeneratedValue`** — Auto-increment primary key
- **`@Column`** — Set constraints like `nullable`, `length`
- **`@Transient`** — Exclude a field from DB persistence
- **`JpaRepository`** — Auto CRUD without writing SQL
- **`Optional` + `orElseThrow()`** — Safe DB lookups with exception handling
- **Interface + Impl pattern** — Proper service layer separation
- **`ddl-auto=update`** — Auto schema management by Hibernate
- **`show-sql=true`** — View generated SQL in console for debugging

---

## 📝 Notes

- `spring.jpa.show-sql=true` is enabled — you can see all SQL queries Hibernate fires in the console, which is great for learning.
- The `tmp` field in `Employee.java` uses `@Transient` — it is **not** saved to the database. It was added to demonstrate the `@Transient` annotation.
- For production use, change `ddl-auto=update` to `ddl-auto=none` and manage schema migrations with a tool like Flyway or Liquibase.
